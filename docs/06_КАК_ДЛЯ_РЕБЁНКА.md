# 🧒 Как для ребёнка — самые простые объяснения

> Этот файл — для случаев, когда «непонятно вообще ничего». Объясняем по-человечески.

---

## Что такое вся эта штука

Представь, что ты придумал **сделать домик для кошки**.

Чтобы его собрать, нужны:
- 🪵 **Доски** (детали)
- 🔨 **Молоток и пила** (инструменты)
- 📐 **Чертёж** (инструкция)
- 👷 **Мастер** (тот, кто соберёт)

С компьютерными проектами то же самое:
- 🪵 Доски = **код**
- 🔨 Молоток = **программы** (Python, Docker, Git)
- 📐 Чертёж = **CLAUDE.md** и **профиль**
- 👷 Мастер = **AI** (Клод)

Вы — **заказчик домика**. Говорите «хочу так и так», и больше ни о чём не думаете. Всё остальное — AI.

---

## Зачем столько правил (CLAUDE.md)

Представь, что мастер делает домик первый раз. Если ему не сказать «не используй гвозди длиннее 3 см, иначе пробьёшь кошку» — он не догадается.

CLAUDE.md = **инструктаж для мастера**. Каждое правило — это «не делай так, потому что...».

Главное правило: **«сначала прочитай свежую инструкцию по инструменту, потом работай»** (это preflight-research). Если мастер взял старую инструкцию по молотку — может ударить молотком не туда. С AI то же самое: если он опирается на старые знания — может выдать неработающий код.

---

## Зачем три режима (Chat / Cowork / Code)

Это как три разных мастерских:

**📚 Chat** — это **библиотека**. Туда ходят узнать «как делают табуретки в Швеции», «что такое шуруп с потайной головкой». Не строим, только читаем.

**🗂 Cowork** — это **прораб**. Он у вас спрашивает «что хотим построить», рисует план, потом отдаёт чертёж мастеру. Сам мало что делает руками, но всё контролирует.

**🔨 Code** — это **мастер с инструментами**. Берёт чертёж и реально пилит, забивает, прикручивает. Главный исполнитель.

**Как пользоваться**:
- Просто захотел узнать что-то → Chat
- Думаешь над идеей и хочешь план → Cowork
- План есть, надо реально написать программу → Code (но обычно Cowork сам отправит туда работу через Dispatch)

## Что такое hooks (зацепки)

Hook — это **датчик в умном доме**. Например, открыли дверь → загорелся свет. Hook в коде: AI хочет создать большой файл → датчик «препятствие!» проверяет, есть ли у тебя research, и если нет — блокирует.

Это **защита от твоих собственных ошибок**, не наказание. Как ремень безопасности в машине.

---

## Что такое MCP

MCP = это **переходник** между AI и другими программами.

Представь: AI живёт в Claude Desktop, а ваши тендеры — на сайте zakupki.gov.ru. AI не может сам туда залезть. MCP — это **трубка-переходник**: «AI, вот тебе доступ к GitHub», «AI, вот тебе доступ к базе данных».

Каждый MCP = одна трубка к одному сервису.

---

## Что такое Docker

Docker = **транспортная коробка для программ**.

Раньше: чтобы запустить базу данных, надо было её установить на свой компьютер, настроить, и она там жила вместе со всем остальным. Если что-то ломалось — всё ломалось.

С Docker: база данных живёт в **отдельной коробке**. Если что-то пошло не так — выкинул коробку, поставил новую за 30 секунд. Внутрь компьютера ничего не попало.

`docker compose up` = «открой коробки и запусти всё что внутри».
`docker compose down` = «закрой коробки и убери на полку».

---

## Что такое git

Git = **машина времени для файлов**.

Каждый раз когда сохраняешься (`git commit`) — git делает **точку сохранения**, как в игре. Если что-то сломалось — можно вернуться к любой точке.

GitHub = **облачное хранилище** для этих точек сохранения. Чтобы не потерять, если ноут сломается.

PR (pull-request) = **запрос на проверку**: «эй, посмотри мои изменения, всё ок?». В нашей системе PR делает AI, а вы просто смотрите и нажимаете Merge.

---

## Что такое .env

.env = **тайная записная книжка с паролями**.

Все важные ключи (пароль к базе, токен GitHub, ключ Anthropic API) лежат в одном файле. Этот файл **никогда не уходит на GitHub** (защищён .gitignore — список запретов).

Если потерял .env — заново сгенерируй ключи и впиши.
Если случайно загрузил .env на GitHub — **сразу** отозвать все ключи и заново.

---

## Что такое preflight (полёт перед полётом)

Preflight = **проверка перед взлётом самолёта**.

Перед каждым серьёзным куском работы AI:
1. Ищет — может, **кто-то уже это сделал** (готовая библиотека)
2. Читает **свежую инструкцию** по нужным инструментам
3. Записывает что нашёл — в `RESEARCH_LOG.md`

Это занимает 5-10 минут. Но **экономит часы** на исправлении того, что AI сделал бы по устаревшим знаниям.

---

## Что такое БОЛЬ/

Это **корзина «потом разберу»**.

В жизни у вас наверняка есть полка или папка, куда сваливается всё — чеки, заметки, листочки. Раз в неделю разгребаете. Тут то же самое, только разгребает AI.

Положили туда сырую идею → сказали «разбери БОЛЬ» → AI сам решит:
- Это идея для нового проекта → сделать чертёж (spec)
- Это улучшение существующего проекта → завести задачу (issue)
- Это просто полезная ссылка → положить в заметки
- Это непонятно/дубль → в архив

---

## Что такое ntfy.sh

ntfy.sh = **«секретное радио» из интернета на ваш телефон**.

У вас есть **секретный канал** (длинный URL). Кто его знает — может в него «крикнуть» (отправить уведомление). Вы подписаны в приложении — получаете все «крики» сразу на телефон.

Зачем: AI ночью увидел что у вас сломался сайт → крикнул в ваш канал → вы утром увидели первое уведомление.

Регистрации нет. Аккаунта нет. Просто URL и приложение.

## Что делать каждый день — самое-самое простое

1. **Утром**: открыли Cowork → посмотрели дайджест (что AI накопил за ночь)
2. **Появилась идея**: написали в Cowork «хочу X» → AI задаёт вопросы → план → AI делает
3. **Появилась смутная мысль**: бросили в `БОЛЬ/inbox/` → когда есть время «разбери БОЛЬ»
4. **Вечером**: посмотрели LAB_JOURNAL — что AI сделал, всё ли ок

Всё. Никаких терминов, никаких команд знать не надо. Просто **говорите с AI как с подрядчиком**.

## Если страшно начать

Это нормально. Любая мастерская поначалу пугает кучей инструментов.

**Самый маленький первый шаг**:

1. Открыть проводник (Win+E)
2. Перейти в папку `C:\HOMEHOMEHOME\ClaudeCombine\ИНФРАСТРУКТУРА\scripts\`
3. Найти файл `setup-wizard.cmd` (с **cmd** на конце, не с ps1!)
4. **Двойной клик** на нём
5. Откроется чёрное окно — это нормально, так выглядит командная строка
6. Идти по шагам, на каждом нажимая `Y` (да) или `S` (пропустить)
7. В конце окно скажет «Готово»

❌ Если кликнули на `setup-wizard.ps1` (без `c` в конце) — он откроется в **редакторе текста**, ничего не запустится. Это особенность Windows. Закройте редактор, кликните на `.cmd` версию.

Потом откройте Cowork и просто скажите: «Привет, я только начал. Расскажи что ты умеешь».

AI ответит и пойдёт с вами рядом. Не надо знать всё заранее.

---

## Почему два файла — `.ps1` и `.cmd`

Один и тот же скрипт лежит в двух обёртках:
- `.ps1` = **сам мозг** (что делать). Не запускается по клику — Windows для безопасности открывает в редакторе.
- `.cmd` = **ручка к мозгу** (как запустить). Двойной клик работает.

**Запускайте `.cmd`. Всегда.** Файл `.ps1` нужен ИТ-людям для тонкой настройки или чтобы посмотреть, что внутри.
