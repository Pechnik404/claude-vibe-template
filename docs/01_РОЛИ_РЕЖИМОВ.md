# Роли трёх режимов Claude Desktop

> Chat — Cowork — Code: у каждого своя работа. Правильное разделение = эффективность.

---

## Chat (значок облачка слева)

**Назначение**: знания, разведка, обучение.

**Когда использовать**:
- «Что такое SQLAlchemy 2.x async?»
- «Сравни FastAPI и Django»
- «Объясни алгоритм Дейкстры на пальцах»
- Любые вопросы без проекта, без файлов, без необходимости что-то менять

**НЕ использовать для**:
- Работы с файлами проекта (нет доступа)
- Изменения кода
- Регулярных задач (нет state)

**Аналог из обычной жизни**: Google + Stack Overflow в одном окне.

---

## Cowork (среднее, значок с задачами)

**Назначение**: оркестратор. Точка входа для всех идей и долгих процессов.

**Когда использовать**:
- Описать новую идею — Cowork задаст уточняющие вопросы, соберёт план
- Создать постоянный Project под идею
- Расписать scheduled-задачи (утренний дайджест, мониторинг CI)
- Открыть Live Artifact (дашборд проекта, KPI)
- Использовать Dispatch (Beta) для параллельной работы агентов

**Главные сущности Cowork**:
- **New task** — разовая задача
- **Projects** — постоянное рабочее пространство (своя папка, свой CLAUDE.md, своя память)
- **Scheduled** — cron-задачи
- **Live artifacts** — HTML-страницы, которые при открытии дёргают MCP и показывают свежие данные
- **Dispatch (Beta)** — отправка работы фоновым агентам

**Аналог**: Jira + Trello + командный chat = единое место для управления работой.

---

## Code (значок `</>` справа)

**Назначение**: исполнитель. Реальная разработка — пишет, рефакторит, тестирует, делает PR.

**Когда использовать**:
- Когда план собран в Cowork и нужно начать реализацию
- Когда нужны hooks, subagents, slash-commands
- Когда работаете с реальным репозиторием
- Глубокий рефакторинг, отладка

**Главные сущности Code**:
- **New session** — стандартная сессия с проектом
- **Routines** — переиспользуемые сценарии (ваши Slash-commands `/ship`, `/security-pass`)
- **Customize** — `CLAUDE.md` уровня проекта
- **MCP** — подключаются через `.mcp.json` в репо

**Аналог**: IDE с автопилотом, как Cursor — но с глубже интегрированными инструментами.

---

## Типичный workflow одной идеи

```
1. Cowork: «Хочу сделать бот для мониторинга 44-ФЗ»
   ↓
2. Cowork: AskUserQuestion × 5 — критерии, регионы, частота
   ↓
3. Cowork: preflight-research → найдено 2 готовых проекта, оценка
   ↓
4. Cowork: план реализации в чат → вы подтверждаете
   ↓
5. Cowork: Dispatch → Claude Code в фоновом worktree
   ↓
6. Code: implement → tests → push в feature-ветку → PR
   ↓
7. Cowork: возврат — отчёт 10 пунктов, ссылка на PR
   ↓
8. Вы: смотрите diff в GitHub, нажимаете Merge
   ↓
9. (Если возник пробел в знаниях во время работы → Chat)
   ↓
10. Cowork: scheduled-задача через сутки — «как себя ведёт бот? логи, ошибки»
```

---

## Правило большого пальца

- **Знание / понимание** → Chat
- **Идея / план / контроль** → Cowork
- **Код / тесты / PR** → Code

Между ними можно переключаться внутри одной сессии (вкладками). Каждый держит свой контекст.

---

## Что НЕ путать

- ❌ Не писать код в Chat — он не сохранит файлы
- ❌ Не задавать общие вопросы в Code — потратите контекст проекта на ерунду
- ❌ Не делать длинную реализацию в Cowork без Dispatch в Code — Cowork лучше как «диспетчер», Code как «исполнитель»
