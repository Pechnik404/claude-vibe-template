# Протокол адаптации шаблона под нюансы проекта

> Главный документ для всех будущих идей. Один шаблон ≠ один и тот же проект.

---

## Аксиома

Каждый проект имеет нюансы. Шаблон `project-template` — это **70% общей основы**, остальные 30% — адаптация под нюансы конкретной идеи.

3 ваших известных проекта (сайт, торговый терминал, 44-ФЗ бот) — это и есть три профиля. Будут появляться новые → новые профили.

---

## Алгоритм адаптации (для каждой новой идеи)

### Шаг 0. Идея в Cowork chat (вы)

Просто пишете идею своими словами. Без структуры. Например:
> «Хочу инструмент, который смотрит на zakupki.gov.ru новые тендеры по моему региону и шлёт мне в телефон, когда подходит под мои критерии.»

### Шаг 1. AI задаёт уточняющие вопросы (AskUserQuestion × 4-7)

С обязательной рекомендацией по каждому вопросу + мировой практикой + адаптацией к РФ. Примеры:
- Какие критерии фильтрации? Сумма от/до, регион, ОКПД?
- Частота опроса? Каждые 10 мин / каждый час / по новостной ленте?
- Канал уведомлений? ntfy.sh / Cowork / email?
- Что делать с старыми тендерами? Хранить / удалять через N дней?
- Юр.лицо? (важно для МАКС, если бы выбрали)

### Шаг 2. AI выбирает профиль

На основе ответов:
- Если есть готовый профиль (например, `gov-44fz-bot.md`) → берём его
- Если нет → создаём новый: `profiles/<short-name>.md` со специфичными нюансами

### Шаг 3. AI запускает preflight-research

Обязательный. См. `02_PREFLIGHT_ГЛАВНОЕ.md`. Результат — в `RESEARCH_LOG.md` нового проекта.

### Шаг 4. AI собирает план в TODO-формате

В чат Cowork — конкретный план с:
- ожидаемыми артефактами (файлы, эндпоинты)
- стоп-условиями (где можем застрять)
- оценкой времени (low/med/high)

Вы подтверждаете или корректируете.

### Шаг 5. AI создаёт репо из шаблона + профиля

```
pwsh scripts\init-project.ps1 -Name <name> -Profile <profile>
```

Это:
- Копирует `project-template/` в новую папку
- Накатывает изменения из `profiles/<profile>.md`
- Заменяет плейсхолдеры (PROJECT_NAME, COMPOSE_PROJECT_NAME) на реальные имена
- `git init` + `git remote add origin`
- Регистрирует scheduled-задачи (если в профиле указаны)
- Подключает project-level MCP

### Шаг 6. AI реализует через Dispatch в Code

Cowork → Dispatch → Claude Code в фоновом worktree.
Cowork в это время свободен — можно говорить о других вещах, проверять live artifact.

### Шаг 7. PR + отчёт 10 пунктов

PR-ссылка в Cowork chat. Вы смотрите diff в GitHub UI, нажимаете Merge сами.

### Шаг 8. Адаптация шаблона по итогам

Если в процессе вылезли нюансы, которых не было в профиле — AI **обновляет профиль** (или создаёт новый). Это и есть **self-improving template** (см. комбо-приём D).

## Что считается «нюансом» (по типам)

### Технические нюансы
- Особая БД (Clickhouse для аналитики, Redis для очередей)
- Real-time требования (WebSocket, SSE)
- Внешние API с особенностями (rate limits, OAuth flow, IP whitelisting)
- Высокая нагрузка (нужен load testing с самого начала)

### Доменные нюансы
- Финансы → строгая идемпотентность, аудит-журналы, налоги (для трейдинга)
- Госзакупки → специфика 44-ФЗ/223-ФЗ, ОКПД2/ОКВЭД2
- Образование → возрастная адаптация UI, доступность WCAG 2.1
- E-commerce → корзина, чекаут, ЮKassa/CloudPayments, фискальные чеки

### Юридические нюансы
- ПДн → 152-ФЗ, хранение в РФ, согласие
- Финуслуги → лицензия ЦБ (если делаете брокерство, не для личного терминала)
- Реклама → ОРД (для рекламной деятельности)

### Среда нюансы
- Один пользователь (вы) vs много
- Public vs private
- Локально vs облако

## Когда создавать новый профиль

Правило **«три раза»**: если три раза подряд проект потребовал одно и то же отклонение от базового — это профиль.

Пример: если в трёх проектах подряд понадобился Redis для кеша — добавить опциональный модуль в `profiles/with-redis.md` или включить в основной шаблон.

## Структура файла профиля

```markdown
# Профиль: <name>

## Когда выбирать
<один абзац>

## Особые предостережения (если есть)
- ...

## Накладывается поверх project-template

### Дополнения к pyproject.toml
### Дополнения к docker-compose.yml
### Структура src/
### Дополнения к .mcp.json
### Дополнения к CLAUDE.md (§15)

## Прочитать перед стартом (preflight обязательно)
- ...

## Готовые проекты на GitHub
- ...

## Тестирование (специфика)
## РФ-специфика
## Юридические нюансы
```

## Главное про адаптацию в одну строку

> **Шаблон фиксирует то, что НЕ должно меняться (правила, защита). Профиль фиксирует то, что меняется по типу проекта. Идея — это разовый разговор.**
