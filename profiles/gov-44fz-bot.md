# Профиль: gov-44fz-bot

> **Когда выбирать**: парсинг zakupki.gov.ru, фильтрация по критериям, уведомления о новых тендерах по 44-ФЗ.

## Что такое 44-ФЗ (короткий контекст)

Федеральный закон №44-ФЗ — «О контрактной системе в сфере закупок товаров, работ, услуг для обеспечения государственных и муниципальных нужд». Все госзакупки публикуются на единой информационной системе **ЕИС** (zakupki.gov.ru). Есть открытый API.

## Накладывается поверх project-template

### Дополнения к pyproject.toml

```
dependencies:
  - httpx              # асинхронные HTTP-запросы к API ЕИС
  - feedparser         # парсинг RSS (часть данных через RSS)
  - lxml               # XML-парсинг (некоторые ответы XML)
  - apscheduler        # планирование периодического опроса
  - pandas             # агрегация по регионам, ОКПД, суммам
```

### Структура src/

```
src/
  zakupki/
    client.py          # API-клиент к ЕИС
    parser.py          # парсинг XML/JSON ответов
    filters.py         # фильтры (регион, ОКПД, сумма, заказчик)
  notifier/
    ntfy.py            # отправка push в ntfy.sh
    digest.py          # формирование ежедневного дайджеста
  storage/             # сохранение найденного в Postgres
  scheduler.py         # APScheduler: опрос каждые N минут
```

## Уведомления — ntfy.sh

Для каждого срабатывания фильтра — push через ntfy.sh:

```
POST ${NTFY_TOPIC_URL}
Title: 44-ФЗ: новый тендер по фильтру "<имя фильтра>"
Body:
  Заказчик: <название>
  Сумма: <X> руб.
  Дедлайн подачи: <дата>
  Регион: <название>
Click action: <URL карточки на zakupki.gov.ru>
Priority: <high для срочных, default для остальных>
```

Один topic-URL = один канал. Можно сделать несколько (например, по приоритету фильтров).

## Тестирование

- Sandbox/тестовые данные ЕИС НЕ предоставляет публично — используем кеш реальных ответов через `pytest-vcr`
- Тесты фильтров — на сохранённых JSON-фикстурах

## Юридические нюансы

- 44-ФЗ + 152-ФЗ: если в системе появятся ПДн (контакты заказчиков, представителей) — включить `ru_pdn: true` и хранить в РФ-регионе
- Антимонопольное — не использовать данные для манипуляций ценами (УК РФ ст. 178)
- Использование для информирования о тендерах = легально и широко практикуется (TenderPro, Контур.Закупки и т.п.)
