# scripts/ — скрипты-помощники

> **Запускать всегда `.cmd` файлы**, НЕ `.ps1`.
> `.cmd` = двойной клик работает, всё открывается в окне.
> `.ps1` = только при ручном вызове из PowerShell (для опытных).

---

## Скрипты

### 🧙 setup-wizard.cmd — мастер первоначальной настройки

**Как запустить**: **двойной клик** на `setup-wizard.cmd` в проводнике.

Что делает:
- Проверяет наличие 10 нужных программ (winget, git, gh, WSL2, Docker, VS Code, Python в WSL, Claude Code, rclone, PowerShell 7)
- На каждом шаге **спрашивает Y/N/Skip** + объясняет ЗАЧЕМ
- Если уже установлено — пропускает автоматически
- Безопасно: ничего не делает без вашего согласия

Когда: **один раз при первом старте** на новом ПК. Потом не нужен.

---

### 📦 init-project.cmd — создание нового проекта

**Как запустить**: **двойной клик** на `init-project.cmd` в проводнике.

Что делает:
- Спрашивает **имя проекта** (например, `my-trading-bot`)
- Спрашивает **тип проекта** (выбор 1/2/3/4)
- Спрашивает **GitHub-логин** (можно пропустить)
- Копирует шаблон → создаёт папку `C:\HOMEHOMEHOME\<имя>\`
- Делает `git init` + первый коммит
- Опционально создаёт GitHub remote через `gh`

Когда: **каждый раз при старте новой идеи**.

#### Через командную строку (для опытных)

```
pwsh init-project.ps1 -Name my-app -Profile web-app -GitHubUser <YOUR_USERNAME>
```

Параметры:
- `-Name` (имя проекта)
- `-Profile` (web-app | trading-terminal | gov-44fz-bot | base)
- `-Target` (куда создать, по умолчанию `C:\HOMEHOMEHOME`)
- `-GitHubUser` (логин GitHub для создания remote)

### 💾 backup-321.cmd — бэкап по правилу 3-2-1

**Как запустить**: **двойной клик** на `backup-321.cmd`.

Что делает:
1. `git push` во все ваши репозитории (копия 1 = GitHub)
2. `rclone copy` в облако (копия 2 = ваш Selectel/Яндекс) — если настроен `-RcloneRemote`
3. zip-архив на втором диске D:\Backups (копия 3 = локально на втором носителе)

Когда: **раз в неделю**, можно по расписанию через scheduled-task.

## Почему .cmd, а не .ps1

В Windows по умолчанию `.ps1` (PowerShell-скрипты) **не запускаются двойным кликом** — открываются в редакторе. Это сделано из соображений безопасности. Чтобы реально запустить, нужно:
1. Открыть PowerShell вручную
2. Возможно разрешить запуск через `Set-ExecutionPolicy`
3. Указать путь к скрипту

Для не-ИТ это барьер. **`.cmd` (батники) работают по двойному клику всегда**, без настроек. Внутри батник вызывает PowerShell с правильными флагами (`-ExecutionPolicy Bypass`) и передаёт ему `.ps1`.

Иными словами: батник — это «двойной клик-обёртка» для PowerShell-скрипта.

## Если что-то не работает

| Симптом | Что делать |
|---|---|
| Окно открылось и сразу закрылось | Запустить через cmd: `cmd → cd C:\HOMEHOMEHOME\ClaudeCombine\ИНФРАСТРУКТУРА\scripts → setup-wizard.cmd` — увидите ошибку |
| «PowerShell не найден» | Перезагрузить ПК после установки PowerShell 7 |
| Кракозябры вместо русского | В окне ПКМ → Свойства → Шрифт → Lucida Console |
| «Доступ запрещён» при git/docker | Запустить cmd от Администратора правым кликом |
