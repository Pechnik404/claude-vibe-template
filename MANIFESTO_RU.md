## Историческая аналогия

**1986–1995. Эпоха Norton Commander.**
Чтобы пользоваться компьютером, нужно было помнить десятки команд. NC был лучшим файловым менеджером своего времени. **Профессионалы боготворили его.** Но 99% людей видели чёрный экран и закрывали крышку ноутбука навсегда.

**1995. Windows 95 пришёл и опустил барьер.**
Мышь, иконки, кнопка «Пуск». NC не стал хуже — он остался для своих. Но рынок ушёл к Win95, потому что **домохозяйки, школьники, бабушки наконец смогли пользоваться компьютером**. Барьер был не интеллектуальный, **барьер был технический**.

**2026. Мы в эре «NC для AI».**
Claude, Cursor, Copilot, GPT — мощнейшие инструменты. Но чтобы реально работать:
- Настроить WSL, Docker, Python, git, gh, MCP-серверы, hooks, .env, ExecutionPolicy
- Помнить разницу `.cmd` vs `.ps1`, UTF-8 BOM vs без BOM, CRLF vs LF
- Разбираться в форматах settings.json, которые меняются от версии к версии
- Знать что Claude не помнит про Claude Design, что pip может потребовать зеркало Yandex, что Telegram-бот в production нужно регистрировать через юрлицо

Тысячи **талантливых людей с гениальными идеями** покупают Claude Pro, открывают, теряются на этапе «как настроить», и через неделю отменяют подписку. Не потому что они глупые. **Потому что барьер технический, как у NC в 1986.**

## Что мы построили

**Boxed-решение** — двойной клик → wizard → новый проект → можно говорить идею.

**0 человеческого участия в технической части:**
- Один `.cmd` файл ставит ВСЁ (Win → WSL → Docker → Python → Claude Code → rclone) с подтверждением каждого шага
- Шаблон проекта со встроенными правилами против дрейфа: hooks реально **блокируют** AI от написания кода без актуальной документации
- 3 готовых профиля под типы проектов (веб, торговый терминал, госзакупки)
- Папка `БОЛЬ/` — корзина для сырых идей, AI сам разбирает на категории
- Документация на русском, **для 10-летнего ребёнка**, без ИТ-жаргона

**Что внутри (53+ файла):**
- Конституция проекта (правила для AI)
- 7 skills, 2 agents, 2 slash-commands
- Hooks на PowerShell с экранированием Windows-нюансов
- Wizard с 10 шагами Y/N/Skip
- Init-project с интерактивным режимом
- Backup по правилу 3-2-1 (GitHub + облако + второй диск)
- Профили под РФ-периметр (ЮKassa, Yandex Cloud, GitVerse, ntfy.sh)
- Reset Protocol для застрявших проектов

## Для кого

**Не для программистов.** Программисты уже умеют пользоваться NC — им наш Windows 95 не нужен.

**Для тех у кого:**
- Есть идея, но нет времени учиться 40 инструментам
- Куплен Claude Pro, но непонятно как им реально работать
- В голове сайт, бот, торговый терминал — но руки опускаются на этапе настройки
- Опыт 8-10 часов в день за компьютером, но не разработчиком
- РФ-юрисдикция (где Stripe не работает, Telegram-боты с подвохом, нужны зеркала pypi)

**Гипотетическая аудитория**: предприниматели, аналитики, инженеры из non-IT отраслей, журналисты-дата, продакт-менеджеры без программистского бэкграунда, талантливые подростки.

## Почему сейчас

3 фактора сошлись в 2026:

1. **AI стал реально способен** (Claude Opus 4.7, Sonnet 4.6) — может вести проект end-to-end если правильно направить.
2. **Anthropic выпустил инфраструктуру** (Plugins, Skills, Hooks, MCP, Frontend Design plugin, Dispatch Beta) — кирпичи есть, дом не построен.
3. **Барьер входа резко обострился** — настройка усложнилась с MCP, hooks, plugins. Без boxed-решения каждый изобретает велосипед.

**Окно возможности: сейчас.** Через год Anthropic либо сделает это сам (и убьёт нишу), либо появятся 10 конкурирующих стартапов (и нишу разделят).
