## ADR-001: Использовать FastAPI + HTMX + Postgres как базовый стек

Дата: 2026-05-23
Статус: Принято (зафиксировано в шаблоне project-template)

### Контекст
Solo-разработчик, до 8-10 ч/день, идеи разной природы (веб, торговый терминал, gov-data-бот). Нужен один стек, покрывающий 70-80% случаев, с минимальной сложностью поддержки.

### Решение
Базовый стек: **Python 3.11+ / FastAPI / SQLAlchemy 2.x async / Postgres / HTMX + Jinja2 / Docker**.

### Альтернативы (отвергнуты)
- Django — больше «батареек», но избыточен для API + HTMX
- Flask — устарел в сравнении с FastAPI (async, type hints, Pydantic)
- Next.js / TS — второй язык, второй процесс, +100% поддержки для solo
- Streamlit — годится для data-демо, не для production-сайтов

### Последствия
- ✅ Один язык (Python) от БД до шаблонов
- ✅ Async по умолчанию
- ✅ Type hints + Pydantic = меньше runtime-ошибок
- ⚠️ HTMX незнаком многим — нужна документация в DECISIONS для каждого нетривиального паттерна
- 🔄 При необходимости React-фронта — добавится через `profiles/web-app-react.md`

### Источники
- pyproject.toml, docker-compose.yml шаблона
- preflight-сессия CLAUDE.md
