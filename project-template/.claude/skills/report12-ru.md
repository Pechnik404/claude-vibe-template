# Skill: report12-ru

> **Назначение**: после каждой задачи печатать в чат краткий русский отчёт из 10 пунктов (по prefs пользователя §12).

## Когда срабатывает

- Завершение задачи (slash-command `/report` или автоматически в конце цепочки)
- Перед маркировкой задачи `completed`

## Шаблон отчёта

```
ОТЧЁТ ПО ЗАДАЧЕ

1. Задача: <одно предложение из формулировки пользователя>
2. Сделано: <одно-два предложения, без воды>
3. Изменено файлов: <N>
   - path/to/file1.py
   - path/to/file2.py
4. Создано файлов: <N>
   - path/to/new1.py
5. НЕ менялось (защита от leftover):
   - path/to/related-but-untouched.py — проверено, не требовалось
6. Тесты: <N passed, N failed> (ступени Pyramid: ✅✅✅✅✅)
7. Confidence: <%> — <одна фраза почему>
8. Риски / TODO:
   - <если есть, иначе "нет">
9. PR: https://github.com/.../pull/<N>  (статус: open / ready for review)
10. Следующий шаг (моя рекомендация): <одно предложение>
```

## Правила

- Только эти 10 пунктов, в этом порядке. Никаких «дополнительно», «также», «P.S.».
- Длина — не более 25 строк.
- Confidence ниже 80% — обязательно объяснить причину в п.7.
- Внутренние логи AI, стек-трейсы — НЕ в отчёте. Они в `LAB_JOURNAL.md`.
- Если задача была «обсуждение» / «research» без кода — пп. 3-6, 9 заменяются на «не применимо».

## Что НЕ включать

- ❌ Маркетинговые формулировки («успешно», «отлично», «всё готово»)
- ❌ Извинения за ошибки (если они были — в п.8 «Риски»)
- ❌ Похвалу пользователю
- ❌ Эмодзи (кроме ✅/❌ в п.6 для статуса ступеней)
