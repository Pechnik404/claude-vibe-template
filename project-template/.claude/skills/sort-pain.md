# Skill: sort-pain

> **Назначение**: разобрать содержимое `БОЛЬ/inbox/` — сырые мысли пользователя — на 4 категории и разложить по местам.
>
> **Когда срабатывает**: пользователь говорит «разбери БОЛЬ», «что в инбоксе», «sort-pain».

---

## Алгоритм

### Шаг 1. Прочитать всё из inbox/

```
ls C:\HOMEHOMEHOME\ClaudeCombine\ИНФРАСТРУКТУРА\БОЛЬ\inbox\
```

Для каждого файла прочитать содержимое (текст, скриншоты — через vision, аудио — через транскрибер).

### Шаг 2. Классифицировать в одну из 4 категорий

**🟢 ИДЕЯ для нового проекта**
Признаки:
- Описывает новую функциональность («хочу сделать бот, который...»)
- Упоминает целевую аудиторию или пользу
- Достаточно крупная (минимум на день работы)

Действие:
- Создать папку `sorted-out/idea-<short-slug>/`
- Положить туда:
  - `original.<ext>` — копия исходника
  - `spec.md` — структурированное описание (на основе шаблона SPEC)
- В отчёте: предложить «Запустить `init-project.ps1` под эту идею?»

**🟡 ДОПОЛНЕНИЕ к существующему проекту**
Признаки:
- Упоминает существующий проект или функцию из него
- Короткое («добавить кнопку», «починить ошибку», «улучшить X»)

Действие:
- Через GitHub MCP создать issue в нужном репозитории
- Заголовок issue — короткая суть
- Тело — оригинальный текст + ссылка на дату из inbox
- В отчёте: ссылка на созданный issue

**🔵 ПОЛЕЗНАЯ ЗАМЕТКА / источник**
Признаки:
- Ссылка на статью, видео, библиотеку
- Цитата из чужого выступления
- Технологический инсайт без привязки к проекту

Действие:
- Если относится к конкретному проекту → дописать в его `DECISIONS.md`
- Иначе → переместить в `sorted-out/notes/<тематика>/`
- В отчёте: куда положено

**⚪ НЕПОНЯТНО / ДУБЛЬ / устарело**
Признаки:
- Невозможно классифицировать
- Совпадает с уже разобранным
- Эмоциональный сброс без действия («задрало X»)

Действие:
- Переместить в `archive/<YYYY-MM>/` с префиксом причины (`dup-`, `unclear-`, `vent-`)
- НЕ удалять — пользователь может вернуться
- В отчёте: «N файлов в архив, гляньте на досуге если интересно почему»

### Шаг 3. Очистить inbox/

Все разобранные файлы должны переехать. `inbox/` остаётся пустым (только `.gitkeep`).

### Шаг 4. Отчёт пользователю

Шаблон:

```
📥 Разобрал inbox/ — <N> файлов:

🟢 ИДЕИ (создал spec'и):
  • <name> → sorted-out/idea-<slug>/spec.md
    Краткая суть: <одно предложение>
    Предлагаю: запустить init-project.ps1 -Name <slug> -Profile <угаданный>

🟡 ДОПОЛНЕНИЯ к проектам:
  • <name> → issue #<N> в <repo>: <ссылка>

🔵 ЗАМЕТКИ:
  • <name> → DECISIONS репо <repo> / sorted-out/notes/<topic>/

⚪ В АРХИВ:
  • <N> файлов — причины: <dup, unclear, vent>

Inbox чист. Готов к следующей порции.
```

---

## Правила безопасности

- **НЕ удалять** ничего — только перемещать
- При сомнениях по классификации — спросить пользователя через `AskUserQuestion`, не угадывать
- Никогда не отправлять содержимое inbox в публичные сервисы (только локально + ваш GitHub)
- Если в файле обнаружены секреты (токены, пароли) — НЕ создавать issue, спросить пользователя

---

## Что НЕ делает skill

- ❌ Не создаёт код по идее (это отдельный шаг через init-project)
- ❌ Не публикует issues в публичные/чужие репо (только ваши)
- ❌ Не редактирует оригинальные мысли — копирует as-is
