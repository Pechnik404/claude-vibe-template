# Slash-command: /security-pass

> **Назначение**: security-аудит текущего состояния проекта или конкретного diff.

## Что делает

1. **Статический скан**:
   - `bandit -r src/` — известные python-уязвимости
   - `detect-secrets scan` — секреты в коде/конфигах
   - `pip-audit` — известные CVE в зависимостях
   - `safety check` — дополнительно по PyPI Safety DB

2. **Анализ конфигурации**:
   - `.env.example` не содержит реальных значений (только плейсхолдеры)
   - `docker-compose.yml` — порты не наружу без надобности, нет привилегированных контейнеров
   - `nginx.conf` (если есть) — security headers (CSP, HSTS, X-Frame-Options)
   - CORS — не `*` в production

3. **Анализ кода через `code-reviewer-ru`**:
   - SQL injection возможности
   - XSS в шаблонах
   - CSRF на формах
   - Path traversal в работе с файлами
   - Insecure deserialization (`pickle`, `yaml.load` без SafeLoader)
   - Race conditions в критичных операциях

4. **РФ-специфика** (если `ru_pdn: true`):
   - ПДн хранятся в РФ-регионе (проверка docker-compose / Terraform)
   - Логи не пишут ФИО, паспорта, телефоны в открытом виде
   - Шифрование at-rest для критичных полей

## Формат отчёта

```
SECURITY AUDIT
Дата: 2026-XX-XX
Проект: <name>

🔴 КРИТИЧНО (исправить немедленно):
1. <файл:строка> — <CWE-XXX: описание>
   Как починить: <конкретный шаг>

🟠 ВЫСОКОЕ (до релиза):
2. ...

🟡 СРЕДНЕЕ (по плану):
3. ...

🟢 НИЗКОЕ (по возможности):
4. ...

📊 СТАТИСТИКА:
- bandit: <N issues>
- secrets: <N found>
- CVE в зависимостях: <N>
- code review: <N security flags>

Общий уровень безопасности: 🔴 Критичный / 🟠 Требует внимания / 🟡 Приемлемо / 🟢 Хорошо
```

## Когда запускать

- Перед первым публичным релизом
- Перед добавлением функционала, работающего с ПДн или платежами
- Раз в месяц для активных проектов (через scheduled-task)
- После обновления зависимостей
