# ntfy.sh — настройка уведомлений за 5 минут (для не-ИТ)

> **Что это**: бесплатный сервис мгновенных пушей на телефон и ПК. Никаких регистраций, без аккаунтов, без API-ключей.
> Работает в России без ограничений.

---

## Шаги (5 минут)

### Шаг 1. Придумать топик-URL (это и есть «адрес» вашего канала)

Топик — это просто длинная строка-секрет в URL. Кто знает URL — тот получает пуши. Поэтому URL должен быть **длинным и случайным**.

Пример хорошего URL:
```
https://ntfy.sh/pechnik404-claude-2026-x7k9q2-alerts
```

❌ Плохо: `https://ntfy.sh/test`, `https://ntfy.sh/myalerts` — кто угодно может писать вам и слышать.
✅ Хорошо: имя+проект+случайные символы.

**Запомните этот URL** — он вам нужен в двух местах: в `.env` и в приложении на телефоне.

### Шаг 2. Установить приложение на телефон

- **Android**: Play Store → «ntfy» (от Philipp Heckel) — бесплатно
- **iOS**: App Store → «ntfy» — бесплатно
- **ПК Windows**: можно использовать веб-версию ntfy.sh в браузере, либо desktop-приложение (есть сторонние)

Открыть приложение → `+` → `Subscribe to topic` → ввести **полный URL** (или только хвост `pechnik404-claude-2026-x7k9q2-alerts`).

Готово, теперь все пуши на этот топик придут вам.

### Шаг 3. Проверить

В браузере или curl:
```
curl -d "Тест ntfy 🎉" https://ntfy.sh/pechnik404-claude-2026-x7k9q2-alerts
```

На телефон должно прилететь уведомление в течение 1-2 секунд.

### Шаг 4. Положить URL в .env

В файл `.env` каждого проекта:
```
NTFY_TOPIC_URL=https://ntfy.sh/pechnik404-claude-2026-x7k9q2-alerts
```

(Уже есть в `.env.example` шаблона)

### Шаг 5. Готово — AI может слать вам пуши

В scheduled-задачах, hooks, скриптах — простой HTTP POST на этот URL.

## Уровни приоритета

ntfy поддерживает 5 приоритетов:
- `1` (min) — тихо, без звука
- `2` (low)
- `3` (default) — стандарт
- `4` (high) — звук + вибрация
- `5` (max/urgent) — повторный звон, минует «не беспокоить»

Передаются через заголовок:
```
curl -H "Priority: high" -d "🔴 CI упал" $NTFY_TOPIC_URL
```

Используем:
- `5` — упавший production
- `4` — упавший CI, требуется внимание
- `3` — обычные алерты
- `2` — еженедельные дайджесты

## Self-hosted (опционально, для большей приватности)

ntfy.sh — open source. Можно поставить на ваш Selectel VDS:

1. На VDS: `docker run -d -p 80:80 binwiederhier/ntfy serve`
2. Указать в DNS поддомен → IP вашего VDS
3. В приложении подписаться на `https://ntfy.yourdomain.ru/your-topic`

Это даёт:
- Полная приватность (данные не идут через ntfy.sh публичный)
- Свой rate limit (а не общий)
- Работает офлайн внутри сети

Для начала — публичный ntfy.sh более чем достаточно. Self-hosted — когда захочется.

## Что НЕ хранить в Topic

Топик — это секрет, но всё-таки публичный сервис. **Не отправляйте через ntfy.sh**:
- Пароли, токены, ключи
- ПДн пользователей (если они есть)
- Коммерческие тайны

Шлите только **факт события** + ссылку. Детали — в защищённое хранилище (БД, GitHub Issues).
